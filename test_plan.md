# 시그니처 자동 정제 모달 UX 개선

현상: 사용자가 "시그니처 자동 정제 (LLM)" 모달을 처음 열었을 때, 첫 번째 그룹 정도만 선택되어 있고 나머지 그룹들은 모두 "통일(대표) 구역 제외됨" 상태로 비활성화(회색) 처리되어 보이는 문제 발생.
원인: 데이터 페칭 로직에서 모든 클러스터를 초기에 `selected: false, isLoading: true` 로 렌더링하고 백그라운드에서 병렬 구동함. 그런데 UI에서는 `isLoading` 상태를 고려하지 않고 `!cluster.selected`일 때 무조건 "제외됨" 오버레이를 덮어씌워서 로딩 스피너를 가림.

## 작업 계획
1. `SignatureRefinerModal.tsx`의 오버레이 및 스타일 조건에 `!cluster.isLoading` 추가
2. 좌측 체크박스와 배지(달력 라벨)에 로딩 중 상태(예: 'LLM 분석 진행 중...') 표시
3. 로딩 중에 체크박스를 비활성화(`disabled`) 처리
