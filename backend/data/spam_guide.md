# Spam Detection Guide
## Authoritative Rules for LLM Spam Classification
Version: v1.8  
Status: Active (Single Source of Truth)

---

## 0. 목적 (Purpose)

본 문서는 통신사/메시징 환경에서의 **스팸(SPAM) 판정 기준을 단일 소스(Single Source of Truth)**로 정의한다.

- LLM은 **본 가이드에만 근거하여 판단**한다.
- 코드 프롬프트는 절차와 출력 형식만 정의하며,
  **판단 기준을 추가·변경·해석하지 않는다.**
- 본 가이드는 오탐 민원 방지를 최우선 목표로 한다.

---

## 1. 절대 정의 (Absolute Definition)

### 1.1 SPAM 정의
**SPAM이란 다음 목적이 명확히 드러난 메시지**만을 의미한다.

- 유해 목적
- 불법 목적
- 사기 목적
- 어뷰즈(abuse) 목적

위 목적이 **텍스트 또는 확인 가능한 콘텐츠**에서
직접적으로 확인되지 않으면 SPAM으로 판정할 수 없다.

---

### 1.2 HAM 정의
다음은 **즉시 HAM**이다.

- 결제/승인/인증 알림 (시간, 금액, 승인번호 포함)
- 공공/행정/정치 메시지 (시청, 구청, 공단, 의원 등)
- 개인화 1:1 트랜잭션 (수신자 식별 가능, 명확한 이벤트)
- 일반 마케팅/광고/홍보/이벤트 메시지
- 5자 이내의 짧은 메시지

> **원칙**: 도박/성인/사기 등 유해 목적이 확인되지 않으면 HAM이다.

---

### 1.4 외국어 메시지 처리 원칙 (Foreign Language Policy)

- **Safe Clean Foreign**: 한글이 전혀 없고 영문/외국어가 문장 형태로 길게 나열된 경우(40자 이상) `HAM-5`로 판정한다. (일반인은 외국어 메시지에 경계심을 갖기 때문)
- **Obfuscated Foreign (Critical)**: 필터 회피 목적이나 의도 숨기기를 위해 외국어와 숫자, 기호를 교묘하게 섞은 난독화는 **무조건 SPAM**으로 판정한다.
- **Short/Ambiguous Foreign**: 짧은 외국어(인증코드 등)는 패턴으로 판단하지 않고 AI 분석(의도 분석) 단계를 거친다.

---

### 1.3 민원 방지 및 난독화 처리 원칙

- **의도가 불확실하면 HAM** (단, 회피 의도가 없는 경우에 한함)
- 단순 "의심", 통계적 경향만으로는 SPAM 판정 불가
- 단, **"비현실적 보상", "사칭", "긴급성 유발"이 느껴지면 → 피싱/사기 의도로 판단**
- **"30자 이내 난독화"** 해당 시 무조건 SPAM

예시:
  - ❌ "20~5000 4 5 카" → 단순 의미 불명 → HAM
  - ❌ "안녕하세요" → 정상 짧은 문장 → HAM
  - ✅ "벳삼삼 가입" → 도박 의도 명백 → SPAM
  - ✅ "Fre222 시간 2용" → 필터 회피 목적 난독화 → SPAM
  - ✅ "ㅇF 까지! ㅌ" → 의도적 난독화 → SPAM

## 2. HARD GATE (최상위 규칙)

> 아래 규칙은 **모든 판단의 최상위**이며,
> 어떤 보조 신호도 이를 뒤집을 수 없다.

### 2.1 Hard Gate Rules

1. **harm_anchor = false → label은 무조건 HAM**
   - 광고성 의심
   - 사업자 정보 불명확
   - URL / 단축 URL 존재
   - 이미지 링크 포함  
   → 위 요소는 단독 또는 조합으로도 SPAM 근거가 될 수 없다.

2. SPAM 확정 조건
   - `harm_anchor = true` (유해/불법/사기/어뷰즈 목적이 명확)
   - **의도가 매우 명확한 경우** (spam_probability ≥ 0.85): route_or_cta 확인 불필요
   - **의도가 애매한 경우** (spam_probability < 0.85): route_or_cta = true 필요

3. `spam_code` 선택은 **증거 기반**
   - CODE EVIDENCE RULES의 필수 패턴
     **1개 이상 매칭 필수**
   - 매칭 0개 → `spam_code = null`, label = HAM

### 2.2 harm_anchor 판정 원칙

**harm_anchor = true가 되는 의도 유형**:

1. **도박 의도**: 불법 도박/베팅 서비스로 유도
2. **성인/유흥 의도**: 불법 성인 서비스로 유도
3. **사기 의도**: 금전적 피해를 유발할 목적
4. **피싱 의도**: 개인정보/금융정보 탈취 목적
   - 비현실적 보상(주식 증정, 현금 지급, 경품 당첨)으로 클릭/가입 유도
   - 대기업/금융기관/공공기관을 사칭하여 신뢰 획득 시도
   - 긴급성 유발(계정 정지, 보안 위협)로 즉각 행동 유도
5. **불법 금융 의도**: 불법 대출/고금리 유도
6. **회피 의도 (Obfuscation)**: 필터링 시스템 회피 목적
   - **"30자 이내 난독화"** 해당 시 무조건 harm_anchor = **true**
   - 문장 구조 파괴, 의미 없는 단어 분절, 한글/영문/특수문자 무작위 혼합
   - "Fre222 시간 2용", "안/녕/하/세/요" (과도한 분절) 예시 포함
   - **내용이 파악되지 않아도, 회피 시도 자체가 유해 의도(harm_anchor)임**

> **판단 기준**: 특정 키워드가 아닌 **메시지 전체 맥락에서 의도**를 파악
> - 은어, 난독화, 우회 표현도 의도 파악의 근거
> - 고도의 난독화 자체가 **회피 의도**의 증거
> - **"비현실적", "수상함", "의심스러움"을 느꼈다면 → 그 의도가 무엇인지 판단하라**

**harm_anchor = false인 경우** (절대 SPAM 불가):
- 위 5가지 의도 중 **어느 것에도 해당하지 않는** 일반 마케팅/홍보
- **외부 기준**(법령 위반, 광고 표기 누락 등)만으로 판단하는 경우

> **핵심 원칙**:
> - "비현실적 보상 + 외부 링크" → **피싱 의도** → harm_anchor = **true**
> - "일반 마케팅인데 광고 표기 없음" → 의도 불명 → harm_anchor = **false**

## 3. Content Agent 역할 제한

**Content Agent는 텍스트만 분석한다. URL 분석은 URL Agent가 담당한다.**

- URL / 단축URL 존재는 harm_anchor 근거가 **아니다**
- URL 내용(도박/성인/피싱)은 URL Agent가 분석 후 Aggregator에서 병합된다
- Content Agent는 URL을 무시하고 **텍스트 내용만** 판정하라

---

## 4. route_or_cta 판정 원칙

> **SMS 매체 특성**: SMS는 발신번호로 회신 가능하므로 **기본적으로 route_or_cta=true**
> 메시지 내 추가 CTA(URL, 전화번호, 카톡 ID 등)가 있으면 CTA가 더 강화된 것으로 판단

**따라서 harm_anchor=true인 SMS는 route_or_cta 조건을 자동 충족**:
- 의도가 매우 명확(spam_probability ≥ 0.85) → SPAM (route_or_cta 확인 불필요)
- 의도가 애매(spam_probability < 0.85) → route_or_cta=true 이미 충족 → 확률 기반 판단

**CTA 강화 요소** (spam_probability 판단 시 참고):
- 금융/계정/시스템 접근: 입금, 송금, 인증번호, 비밀번호, 앱 설치, 로그인 요구
- 명시적 연락 수단: 전화번호, 카톡 ID, URL 제공

**예시**:
- "벳삼삼" (CTA 없어도 의도 명확) → SPAM
- "비/상/금/비/대/면" (난독화=회피의도, SMS 자체가 CTA) → SPAM
- "마ㄹㅇ 실장" (난독화된 성인 유흥, SMS 자체가 CTA) → SPAM

## 5. CODE EVIDENCE RULES
> spam_code 선택 시 **아래 카테고리 참조**.
> **의도가 명확하면 키워드 매칭 없이도 해당 코드 적용 가능.**
> 의도는 파악되나 카테고리 특정 불가 시 → code "0" (기타 스팸)

### 1) 도박 / 게임 (code "3")
- 도박, 배당, 카지노, 토토, 픽, 바카라, 슬롯 등

### 2) 성인 / 유흥 (code "1")
- 출장안마, 유흥, 룸싸롱, 오피 등
- **구조적 패턴**: 지역명 + 불분명한 상호 + 연락처만 존재
  - 사업자 정보(상호, 주소, 사업자번호) 없이 연락처만 제공
  - 난독화(초성, 숫자→한글) 사용 시 회피 의도로 간주
- 직함 포함 시: 실장/상무/팀장/대표 + 연락처

### 3) 통신 / 휴대폰 스팸 (code "0")
- 인터넷+TV, 현금지원, 번이, 기변, 특가
- 단, 지점/사업자 명확 시 HAM-2 우선

### 4) 대리운전 (code "0")
- 대리운전, 콜대리, 이용후무료 등

### 5) 불법 의약품 (code "1")
- 비아그라, 시알리스, 처방없, 불법약 등

### 6) 금융 / 대출 사기 (code "3")
- 개인돈, 급전, 무서류, 고금리 등
- **구조적 패턴**: 대출 안내 + 금융기관 정보 없음 + 개인 연락처(카톡/전화) 유도
  - 정식 금융사는 은행명/저축은행명/등록번호 명시
  - 개인 카카오톡/휴대폰으로 상담 유도 시 불법 대출 의심
- 고도 난독화(띄어쓰기, 특수문자 삽입) = 회피 의도 = harm_anchor
- 정상 금융 안내(금융사명+공식채널 명시)는 HAM 우선

### 7) 구인 / 부업 (불법·어뷰즈) (code "0")
- 투잡, 재택, 일급, 당일지급
- 합법 구인은 HAM-3 가능

### 8) 나이트클럽 (code "1")
- 나이트, 클럽, 입장권, 테이블

### 9) 주식 리딩 / 사기 (code "2")
- 리딩방, 급등주, 수익인증, VIP방

### 10) 로또 사기 (code "2")
- 로또 번호 공유, 당첨보장, 가입비

### 11) 피싱 / 스미싱 (code "2")
> **의도 기반 판단** - 아래 의도가 파악되면 SPAM
- **사칭 의도**: 대기업/금융기관/공공기관을 사칭하여 신뢰 획득
- **미끼 의도**: 비현실적 보상(주식/현금/경품)으로 클릭 유도
- **긴급성 의도**: 계정 정지/보안 위협으로 즉각 행동 유도
- **정보 탈취 의도**: 개인정보/금융정보 입력 유도

> 예시 의도 조합:
> - "삼성전자 100주 무료" → 사칭 + 미끼 = 피싱 의도
> - "계정이 정지됩니다 확인하세요" → 긴급성 + 정보탈취 = 피싱 의도
