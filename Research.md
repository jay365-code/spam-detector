# 입력 테스트 URL 엑셀 저장 및 파기(Drop) 처리 로직 분석 보고서

이 문서는 스팸 필터 파이프라인에서 입력된 텍스트/이메일로부터 URL을 추출하고, 이를 엑셀(보고서) 및 차단 목록(URL 중복 제거 시트)에 **저장할지 혹은 파기(Drop)할지** 결정하는 핵심 파이썬 코드(`batch_flow.py`, `excel_handler.py`)의 동작 로직을 분석한 결과입니다.

## 1. 개요
스팸 메시지에서 추출된 URL을 무분별하게 차단 목록에 올리면 정상 사이트가 차단되는 False Positive(오탐) 대참사가 발생할 수 있습니다. 이를 막기 위해 파이프라인의 최종 Aggregator 노드에서 AI의 판단 사유와 URL의 형태를 복합적으로 검증하여 **`drop_url = True`** 플래그를 결정합니다. 이 플래그가 켜지면 엑셀 파일 생성 시 해당 URL은 백지화(빈칸) 처리되어 안전하게 차단망에서 제외됩니다.

---

## 2. URL 파기(Drop) 핵심 3대 조건 (`batch_flow.py`)

`batch_flow.py`의 `aggregator_node` 함수 내부에 구현된 주요 Drop 조건(644번째 줄 부근)은 다음과 같습니다. 세 조건 중 하나라도 만족하면 `final["drop_url"] = True`가 선언됩니다.

### 2.1. 진짜 방패막이 (Safe Injection / Decoy) 식별
- **초기 식별 로직:** URL Agent의 판결 사유(`u_reason`) 문장을 소문자로 변환한 뒤, 미리 정의된 키워드가 포함되어 있는지 검사합니다. (`"위장 url"`, `"정상 도메인 위장"`, `"방패막이"`, `"decoy"`, `"safe url injection"`, `"미끼 링크"`, `"필터 회피"`)
- **안전망 검증 로직:** AI가 방패막이로 식별했더라도, 해당 URL이 **실제로 신뢰 구간에 있는 정상 사이트(화이트리스트 등)로 검증(is_confirmed_safe = True)** 된 경우에만 최종적으로 Drop을 승인(`is_injection = True`)합니다.
- **결과:** 단축 URL 서비스의 차단 경고창이나, 스패머가 만든 가짜 기업 소개 페이지 등에서 AI가 실수로 방패막이라고 멍청하게 판단하더라도, 이는 `is_confirmed_safe=False`이므로 파기되지 않고 정상적으로 **블랙리스트(URL 중복 제거 엑셀 시트)에 무사히 보존**됩니다.

### 2.2. 무효한 가짜 IP (Fake IP) 필터링
- **로직:** 정규식(`^\d{1,3}(\.\d{1,3}){3}$`)을 사용해 URL의 형태가 전형적인 IP 주소 배열(예: `1.1.1.1`)인지 확인합니다.
- **결과:** 실제 접속 가능한 도메인이 아닌, 단순 버전명이나 오탐으로 추출된 숫자 나열일 경우 `is_fake_ip = True`를 발동하여 파기합니다. (`drop_url_reason = "empty_or_fake_ip"`)

### 2.3. 파손/찌꺼기 KISA URL 강제 비우기 (Mismatched Extraction)
- **로직:** KISA에서 넘어온 사전 추출 URL(`pre_parsed`)이 본문에 존재하는 실제 스팸 URL 도메인과 완전 다르고 파라미터나 경로도 없는 단독 찌꺼기 문자열인 경우 이를 파기합니다. AI는 본문 속 진짜 URL을 새로 찾아냈기 때문에 오염된 기존 URL 필드를 비워버리는 것입니다.
- **결과:** `is_mismatched_extraction = True`가 되어 파기합니다. (`drop_url_reason = "broken_kisa_extraction"`)

## 3. UI 및 엑셀 단독 도메인 보정 처리 (UI Sync)

엑셀을 처리하는 `excel_handler.py`의 동작 방식과 UI 화면 간의 동기화를 위해, `batch_flow.py`에서는 추가적인 `drop_url` 조건을 강제합니다. (708번째 줄 부근)

1. **`pre_parsed_url` (KISA 원본 URL 파라미터)이 비어있는 경우**
   - 만약 텍스트가 정상이고 악성 URL만 분리된 사례(Red Group)가 아니라면, 엑셀 핸들러 로직 규칙에 따라 URL을 표기하지 않고 비워야 하므로 사전에 `drop_url = True`를 적용합니다. (`empty_pre_parsed_url_sync`)

2. **KISA 원본 URL이 파손된 형태(단순 도메인, 특수문자 포함)인 경우**
   - KISA 원본에 URL이 있더라도, 경로가 전혀 없거나(`is_bare`), 괄호 및 별표(`*, {, }`) 등 부적절한 특수기호가 남아있다면 파손된 데이터로 간주합니다.
   - 단, 해당 URL이 명백한 SPAM 증거나 악성 URL로 확정되었을 때(`final.get("red_group")` 등)는 보존을 허용하고, 그렇지 않으면 일괄 `drop_url = True` 처리합니다. (`bare_or_corrupt_domain_sync`)

## 4. `excel_handler.py`에서의 최종 엑셀 저장 로직

위의 `batch_flow.py`에서 결정된 판정 결과를 바탕으로, 최종적으로 `.xlsx` 파일을 만들 때 적용되는 로직은 다음과 같습니다. (808번째 줄 부근)

- **URL 추출 및 저장 조건:**
  메시지가 SPAM으로 판정되었거나(`is_spam == True`), 악성 URL이 성공적으로 발췌되었을 때(`malicious_url_extracted == True`) 엑셀의 "URL" 열과 "URL중복 제거" 시트에 URL을 기입합니다.
- **`drop_url` 플래그 작동 시 처리 방침:**
  결과 딕셔너리에 `result.get("drop_url") == True`가 활성화된 데이터 행은 `ws.cell(row=row_idx, column=url_col_idx, value="")`를 실행하여 엑셀 **URL 칸을 빈칸으로 강제 덮어쓰기** 처리합니다.
- 이에 따라 **파손된 URL 단축키**, **가짜 IP**, **방패막이(정상 사이트)**는 모두 엑셀에서 완벽히 누락되므로 시스템 블랙리스트에도 등재되지 않게 방어됩니다.

---

## 5. 결론
입력 테스트 URL은 다음의 엄격한 3중 검증망을 거칩니다:
1. **문법적 검증:** IP 형태나 괄호 등 파손된 문자열 제외
2. **AI 의미론적 검증:** "방패막이", "조작" 등 의도가 불순하게 섞인 정상 링크 강제 폐기
3. **원본 정합성 검증:** KISA 입력단에서 잘려온 찌꺼기 문자열은 본문 내 실제 URL과 대조 후 삭제

이 유기적인 플로우를 통해 안전하게 악성 스팸 도메인만을 선별하여 엑셀 및 시스템 블록리스트로 반환하며 안정성을 확보하고 있습니다.
