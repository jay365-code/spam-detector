# 스팸 탐지기 엑셀 결과 파일 1차 검수 검증 리스트 (v1.1)

본 문서는 운영자가 엑셀 파일(MMSC 스팸 추출물 등)의 1차 검수 작업을 마친 후, 결과물에 누락되거나 잘못 적용된 규칙(휴먼 에러)이 없는지 최종 검증하기 위한 가이드라인입니다. `validate_excel.py` 검증 스크립트를 통해 아래 항목들이 1초 만에 자동 교차 검증됩니다.

---

## 1. 육안분석 시트 (컬러 서식 및 무결성 검증)
작업자가 메시지 내용과 배경색, 구분 코드를 올바르게 기입했는지 점검합니다.

- [ ] **항목 1**: `구분` 기호가 'o' (일반 스팸 혹은 Type B) 인접해 있는데, 해당 메시지 셀의 **배경색이 황금색(`FFF2CC`)이 아닌 경우** 위반 (색상 지정 누락 확인)
- [ ] **항목 2**: Reason 컬럼에 `[텍스트 HAM + 악성 URL 분리 감지]` 로그가 존재하는데, 해당 메시지 셀의 **배경색이 핑크색(`FFCCCC`)이 아닌 경우** 위반
- [ ] **항목 3**: 분리 감지(핑크색) 타겟인데, 실수로 `구분` 기호에 'o'를 직접 기입한 경우 위반 (분리 감지 케이스는 구분에 'o'를 비워두는 것이 원칙)

## 2. URL 중복제거 시트 (IP 도메인 검증)
과탐/오류를 발생시킬 수 있는 비정상 URL을 걸러냅니다.

- [ ] **항목 4**: (신규 반영) 도메인이 일반 문자가 아닌 `2.3.4.5` 와 같은 순수 **IP 주소(IPv4) 포맷**인 경우 오탐 우려 및 특수 룰 적용을 위해 위반 적발 (수동 삭제 요망)

## 3. 문자/문장 차단등록 시트 (PDF 가이드라인 스펙 강력 통제)
최신 스팸태깅 수작업 가이드에 명시된 시그니처 길이 제한(데드존 및 하한선/상한선)을 엄격하게 준수합니다.

- [ ] **항목 5 [문자열]**: 시그니처 길이가 **9 bytes 미만**이거나 **20 bytes를 초과**하면 위반 (문자열은 반드시 9~20 바이트 구간이어야 함)
- [ ] **항목 6 [문장]**: 시그니처 길이가 **39 bytes 미만**(21~38바이트의 데드존 포함)이거나 **40 bytes를 초과**하면 위반 (문장열은 오직 39바이트, 40바이트 길이만 허용됨)

## 4. 시트 간 데이터 교차 검증 (Cross Check)
작업 중 행을 수동으로 추가/삭제했을 시, `TRAP.문장 중복제거` 시트 우측 상단에 위치한 <통계 요약표>의 개수값이 갱신되지 않는 휴먼 에러를 교차 적발합니다.

- [ ] **항목 7**: 통계 표상의 **SPAM 분류 개수**가 `육안분석` 메인 시트의 `구분=o` 개수, **`Red Group=O` 개수**, 그리고 **`[텍스트 HAM + 악성 URL 분리 감지]`(핑크색) 개수**를 모두 합산한 총 스팸 발췌 라인 수와 정확히 일치하는가?
- [ ] **항목 8**: 통계 표상의 **URL 추출 숫자**가 `URL중복 제거` 시트에 나열된 (일반 URL + 단축 URL) 전체 데이터 개수와 일치하는가?
- [ ] **항목 9**: 통계 표상의 **문자열/문장열 숫자**가 각각 `문자열중복제거`, `문장중복제거` 시트에 실제로 채워진 행(Row)의 정확한 전체 줄 수와 일치하는가?
