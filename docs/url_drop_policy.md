# 스팸 필터 및 엑셀 URL 보존/Drop 정책 설계서 (URL Drop Policy)

이 문서는 스팸 필터 파이프라인에서 추출된 URL을 엑셀 보고서(및 URL 스팸 차단망)에 보존할지, 아니면 안전을 위해 파기(Drop)할지 결정하는 핵심 논리와 규칙을 정의합니다.

## 1. 개요 및 목적
스팸 메시지에서 추출된 URL은 엑셀 보고서의 [육안 검사 시트]와 보안 장비에 입력되는 [URL 중복 제거 시트(블랙리스트)]의 핵심 데이터로 활용됩니다. 
하지만 스패머의 기만 전술(정상 사이트 위장)이나 텍스트 전처리 과정의 오류로 인해, **무분별하게 모든 URL을 보존하고 차단망에 밀어 넣으면 정상 사이트가 차단되는 대참사(False Positive)가 발생할 수 있습니다.** 이를 방지하기 위해 엄격한 **'URL Drop 규칙'** 이 적용됩니다.

---

## 2. Drop (엑셀 및 차단 블록리스트에서 완전 삭제) 대상 조건

다음 조건 중 하나라도 충족되면 URL 필드는 빈칸 처리(`drop_url = True`)되며, 시스템과 엑셀 리포트에서 제거됩니다.

### 2.1. 방패막이 (Decoy / Safe Injection)
- **정의:** 스패머가 분석 시스템과 보안 필터를 햄(Ham)으로 오인하게 만들기 위해 고의로 끼워 넣은 유명 뉴스 링크, 네이버쇼핑, 공식 홈페이지(예: `samsung.com`) 등의 **완전한 정상 URL**.
- **Drop 이유:** 이 주소가 살아서 액셀의 스팸 목록으로 들어가면, 회사 방화벽이나 스팸 차단 시스템이 정상 사이트들까지 무더기로 불법 사이트로 간주하여 차단하는 **대참사**가 벌어짐. 이를 방지하기 위해 가장 철저하게 Drop 시켜 분석 자체를 배제함.
- **트리거 조건:** AI 요원이 사유서에 방패막이/미스매치 키워드(`"방패막이"`, `"위장 url"`, `"정상 도메인 위장"`, `"decoy"`, `"safe url injection"`, `"미끼 링크"`, `"필터 회피"`) 중 하나라도 언급하고, **동시에 해당 URL이 시스템의 화이트리스트 등에서 실제로 안전한 도메인으로 확정명시(`is_confirmed_safe=True`)** 되었을 때만 최종 파기(Drop)가 발동됩니다.
  - (단축 URL 경고창이나 가짜 기업 페이지 등 스패머가 조작한 URL을 AI가 멍청하게 방패막이라고 부르더라도 안전망 검증을 통과하지 못하면 스팸 블랙리스트에 보존시킵니다)

### 2.2. 파손된 찌꺼기 도메인 (Mismatched Extraction)
- **정의:** KISA 전처리 혹은 정규식 추출 과정 중 문자열이 심하게 훼손/절단되어 파편(예: `daum.net` -> `aum.net`, 특수문자 포함 등)만 남은 무의미한 텍스트.
- **Drop 이유 / 조건:** 이 파손된 문자는 자체로는 접속 불가능한 쓰레기 데이터이므로 **무조건 파기(Drop)** 합니다. 
  - (AI가 본문에서 진짜 숨겨진 원본 주소를 찾아냈더라도, 이를 KISA 원본 대신 덮어쓰기(Override)하지 않고 URL 자체를 비웁니다.)
- **⚠️ 예외 — HAM 메시지의 베어 도메인(Bare Domain):**
  - KISA 원본 URL이 경로(Path) 없이 도메인만 있는 형태(예: `example.com`)더라도, **최종 판정이 HAM(정상 메시지)인 경우**에는 찌꺼기로 간주하지 않고 **원본 URL을 그대로 보존**합니다.
  - **보존 이유:** KISA가 입력한 원본 데이터 자체가 처음부터 베어 도메인 형태로 제공된 경우는 파손이 아닌 정당한 원본입니다. 이를 찌꺼기로 간주하여 Drop하면 오히려 정상 데이터가 소실됩니다.
  - **스팸 판정 시에는 예외 없음:** 동일한 베어 도메인이라도 최종 판정이 SPAM이면 찌꺼기 도메인 Drop 규칙이 그대로 적용됩니다.

### 2.3. 가짜 IP (Fake IP)
- **정의:** `1.1.1.1`이나 `2.0.1.7`처럼 형식만 IP처럼 보일 뿐 실제 접속 가능성이 없는 난독화/가짜 문자 배열.
- **Drop 이유 / 조건:** 물리적 유효성이 없으므로 **무조건 파기(Drop)** 합니다. 찌꺼기 도메인과 마찬가지로 본문 추출 URL로 덮어쓰지 않습니다.

### 2.4. 난독화 파편 URL (Fragment Artifact)
- **정의:** 스패머가 `ⓚo.fm/mK8`처럼 특수문자로 URL을 난독화할 때, 정규식이 앞글자를 잘라먹어 생긴 파편(예: `o.fm/mK8`). 원래 의도된 URL(`ko.fm/mK8`)과 동시에 추출되어 블랙리스트에 중복 등재됩니다.
- **Drop 이유:** 의도하지 않은 찌꺼기 URL이 필터에 등록되면 전혀 무관한 도메인(`o.fm`)에 대한 오탐(False Positive)을 유발합니다.
- **트리거 조건:** 아래 세 가지를 **모두** 만족할 때만 파편으로 판정하여 `valid_extracted_urls`에서 제거합니다.
  1. URL Agent가 접속에 성공한 `extracted_url`이 존재 (사실 기반 안전장치)
  2. URL A가 URL B의 **끝부분(suffix)**과 일치 (예: `o.fm/mK8`은 `ko.fm/mK8`의 suffix)
  3. 잘려나간 접두사에 `.`이 **없음** (서브도메인 관계가 아닌 단순 글자 잘림)
- **⚠️ 보존 예외 — 정상 서브도메인:** `blog.naver.com`과 `naver.com`처럼 접두사가 `.`으로 끝나는 정상적인 서브도메인 관계는 파편이 아니므로 보존됩니다.

> 💡 **[시스템 아키텍처 관점의 파기 원칙]**
> 분류된 스팸 메시지 본문은 **ML 학습용**으로 사용되며, URL과 시그니처는 각각 **독립적인 차단 필터 시스템**에서 사용됩니다. 
> 따라서 KISA 입력 URL 자체가 정상적이지 않은 경우(찌꺼기, 가짜 IP), 억지로 본문의 다른 URL로 오버라이드해서는 안 됩니다. **비정상 URL은 즉각 파기(Drop)하고, 대신 시그니처(문자열)를 추출하여 시그니처 기반 필터링 시스템이 해당 스팸을 차단하도록 작동시키는 것**이 올바른 아키텍처 설계입니다.

## 3. Drop 금지 (엑셀에 무조건 끝까지 보존) 대상 조건

아무리 접속이 불가능하거나 상태가 이상하더라도, 아래의 경우엔 **원문의 증거로써 엑셀에 절대 지우지 않고 그대로 보존**합니다.

### 3.1. 데드링크 (접속 불가 / Dead Domain)
- **정의:** 추출된 주소가 DNS 에러가 발생하거나 서버가 죽어서 더 이상 접속할 수 없는 사이트.
- **보존 이유:** 방패막이도 아니고 찌꺼기도 아니라면, 이 도메인은 사기꾼들이 돈을 주고 샀다가 방금 전 당국에 의해 압류/차단된 **진짜 불법 스팸 도메인**일 확률이 100%에 가깝습니다. 비록 지금은 접속이 안 되더라도, 다시 살아날 가능성이 있고 보안 관리자가 사기꾼의 패턴을 추적해야 하므로 **스팸 블랙리스트에 올리기 위해 무조건 엑셀에 보존합니다.**
- **⚠️ 예외 — 베어 도메인(Bare Domain)인 경우:**
  - 데드링크라도 URL이 경로(Path)나 쿼리 없는 **베어 도메인 형태**(예: `example.com`)이면 **Drop** 처리합니다.
  - **이유:** URL과 시그니처는 즉시 차단 필터로 사용됩니다. 베어 도메인을 블랙리스트에 등록하면 해당 도메인 전체 사용자가 차단되는 **False Positive 참사**가 발생합니다. 경로가 없으면 특정 스팸 패턴 추적도 불가능합니다. 이 경우 URL 차단 대신 **IBSE 시그니처 기반 차단**에 위임합니다.

## 4. 백엔드 시스템 구현 포인트 (`batch_flow.py`)
파이프라인의 종단(`aggregator_node`)에 도달한 최종 종합 데이터에서 위의 논리가 실행됩니다.
- 선행 AI 모델들이 판별한 결과와 사유(Reason) 문장을 소문자로 변환한 뒤 키워드를 스캔하여 `is_injection`, `is_mismatched_extraction`, `is_fake_ip` 플래그를 정교하게 켭니다.
- 최종적으로 `drop_url = True`가 활성화된 케이스만이 `excel_handler.py`에서 빈칸으로 삭제 처리되며 차단망을 보호하게 됩니다.
